# AGENTS.md

## 報告の書き方

- 最初に結論を書く。
- ユーザーが求めている答え、実施結果、次に必要な行動を優先する。
- 作業ログのような細かい経過説明を長く書かない。
- できなかったことがある場合は、理由と次の選択肢を短く書く。
- 不要な前置き、言い訳、自己言及を避ける。

## 情報量の調整

- ユーザーが短い答えを求めているときは、短く答える。
- 判断材料が必要なときだけ、根拠や注意点を足す。
- コード、コマンド、ファイル名は必要なものだけ示す。
- 箇条書きは読みやすくなる場合に使い、項目を増やしすぎない。

## 避ける表現

- 「以下に詳細を説明します」のような中身の薄い導入。
- 「実装可能性の観点から検討を行いました」のような硬い言い回し。
- 「サポートします」「提供します」など、英語を直訳しただけの表現。
- 「設定変更対応方針確認」のように漢語だけを連ねた表現。
- 結論より先に背景や経緯を長く説明する書き方。

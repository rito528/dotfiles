# AGENTS.md for config/agents/

このドキュメントは `config/agents/` 配下を編集するときの運用ルールを定義します。  
共通ルールは `config/AGENTS.md` を前提とし、このファイルでは AI Agent スキル定義に固有の注意点だけを扱います。

## 1. このディレクトリの責務

- `config/agents/skills/` は home-manager 配備対象の AI Agent スキル定義の正規配置
- ここで管理する内容はローカル実験用ではなく、`~/.claude/skills` と `~/.agents/skills` に反映される共有資産

## 2. 編集方針

- 追加するスキルは、複数プロジェクトで再利用できるものに限定する
- 一時的な検証手順や個人メモは置かない
- 特定リポジトリ前提の強い手順を書く場合は、ここではなく `.agents/skills/` など別の置き場を検討する

## 3. 安全性の観点

- 権限昇格、破壊的操作、秘密情報へのアクセスを無造作に促す記述を避ける
- コマンド実行の許可判断に踏み込む場合は、ルート `AGENTS.md` と `config/claude/settings.json` の方針と矛盾させない
- ネットワーク依存やローカル固有パス依存の強い内容は、グローバル配備対象にしない

## 4. レビュー観点

- グローバル配備しても誤用しにくいか
- 手順が過度に長くなく、前提条件が明示されているか
- 他の AI Agent から見ても解釈しやすい文面になっているか
- `config/agents/skills/README.md` の説明と矛盾していないか

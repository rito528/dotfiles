# git commit スキル

## コミットメッセージ

- コミットメッセージは、コンベンショナルコミットに従うこと
- コミットメッセージは、差分を読めばわかるコメントではなく、なぜそのような変更をしたのかを書くこと
- プロジェクト単位のルールや指示がない限り、コミットメッセージは日本語で記述すること
- Claude がコミットする場合、Co-Author として Claude を追加すること

## コミット署名

- コミット署名はコミットをするたびに要求する設定がデフォルトとなっている
  - AI Agent がコミットをする際は、署名を**その時に限り**要求しないオプション `--no-gpg-sign` を使用すること

## コミット時のブランチ

- メインブランチ(例: main, master)ではなく、作業ブランチでコミットすること
- 現在のブランチがメインブランチである場合、コミット前に作業ブランチを作成して切り替えること
- ブランチの切り替えには `git checkout` ではなく `git switch` コマンドを使用すること
  - 新規ブランチ作成と切り替えは `git switch -c <branch-name>` を使用すること
  - 既存ブランチへの切り替えは `git switch <branch-name>` を使用すること

## コミットの条件

- 変更内容がまとまったタイミングで行い、コンパイルやテストが通っている状況で行うこと
  - コンパイルやテストの失敗原因がそのコミットの内容に関係ない場合は、ユーザーにその旨を伝え、指示を仰ぐこと

## スコープ

- このスキルはコミットのみを行う。PR の作成は行わないこと
- **単独で呼び出された場合のみ**、コミット完了後に PR 作成を提案したり、確認を求めたりしないこと
- 別のスキル（例: create-pr）からサブルーチンとして呼び出された場合は、呼び出し元の指示に従うこと

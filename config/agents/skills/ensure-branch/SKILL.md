# ブランチ確認・切り替えスキル

コミットや PR 作成の前に、現在のブランチが作業内容に対して適切かどうかを確認し、必要であれば切り替える。

## 判断基準

以下のいずれかに該当する場合、適切なブランチへ切り替える:

1. **メインブランチにいる場合** (例: `main`, `master`)
2. **ブランチが変更内容と明らかに無関係な場合**:
  - 変更内容とブランチ名が全く関係ない（例: `feat/add-pr-skill` ブランチで全く別の機能を実装している）
  - 以前の作業で使ったブランチが残っていて、新しい作業とは無関係

判断が難しい場合はユーザーに確認すること。

## 手順

### 1. 現在のブランチを確認する

```bash
git branch --show-current
git log --oneline -5
```

### 2. 切り替えが必要か判断する

- 上記「判断基準」に照らして、切り替えが必要かどうかを判断する
- 変更内容（`git diff`・`git status`）を参照してブランチ名の適切さを評価する

### 3. 必要であれば切り替える

変更内容から適切なブランチ名を生成して切り替える（例: `feat/add-pr-skill`, `fix/ci-path`）:

```bash
git switch -c <branch-name>
```

既存ブランチへ切り替える場合:

```bash
git switch <branch-name>
```

**注意**: ブランチの切り替えには `git checkout` ではなく `git switch` を使用すること。

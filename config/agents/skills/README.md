# config/agents/skills/

`config/agents/skills/` は home-manager 配備対象の AI Agent スキル定義です。  
このディレクトリの内容は [modules/claude.nix](/home/rito528/dotfiles/modules/claude.nix:1) と [modules/agents.nix](/home/rito528/dotfiles/modules/agents.nix:1) によって `~/.claude/skills` と `~/.agents/skills` へ反映されます。

このディレクトリを編集する前に、`config/AGENTS.md` と `config/agents/AGENTS.md` も参照してください。

## このディレクトリの意味

ここに置くスキルは、リポジトリローカルのメモや一時的な実験ではありません。  
日常的に使うグローバルな AI Agent スキルとして配備される前提で管理します。

## 配置基準

このディレクトリに置くのは、次の条件を満たすスキルに限ります。

- 複数プロジェクトや複数タスクで再利用できる
- 個人の一時的な検証ではなく、継続運用する価値がある
- 権限昇格、破壊的操作、秘密情報へのアクセスを不用意に促さない
- スキルの説明だけで、利用者が期待する挙動を概ね理解できる

## 置かないもの

- 特定リポジトリ専用の一時的ワークフロー
- 検証中の未整理スキル
- 個人メモに近い断片的な手順
- シークレットやローカル固有パスへの依存が強いもの

そのようなものは、必要に応じてリポジトリローカルの `.agents/skills/` 側で管理することを検討してください。

## レビュー観点

- グローバル配備されても誤用しにくいか
- 他のリポジトリでも破綻しない表現になっているか
- 権限判断や危険操作の扱いが `AGENTS.md` の原則と矛盾していないか
- 手順が長すぎず、利用者に過剰な前提知識を要求していないか

## 運用メモ

このディレクトリの変更は「このリポジトリに追加しただけ」とは見なさず、ホーム環境に配備される共有資産の更新として扱ってください。  
スキルを追加する際は、ローカルで便利かどうかだけでなく、グローバル反映して問題ないかを先に判断します。
